## Overview

The Hospital Doom Loop Model Interactive Dashboard is a browser-based visualisation environment developed to accompany the *Hospital Doom Loop Model* System Dynamics (SD) study. The dashboard reproduces the principal dynamic behaviours generated by the underlying Vensim simulation model whilst providing an accessible and interactive interface for exploring the evolution of key hospital-system variables over time.

The dashboard was designed to support:
- Research communication
- Policy discussion
- Teaching and demonstration
- Interactive interpretation of System Dynamics outputs

The interface is implemented using:
- HTML
- JavaScript
- Plotly.js

No additional software installation is required beyond a modern web browser.

---

## Model Purpose

The dashboard visualises the reinforcing congestion dynamics described in the accompanying paper concerning post-pandemic hospital pressures and the persistence of healthcare-system overcrowding.

The model captures interacting feedback processes involving:
- Elective waiting-list growth
- Rising patient complexity
- Longer lengths of stay
- Discharge delays
- Bed occupancy pressure
- Workforce burnout

These interacting mechanisms collectively produce the persistent congestion behaviour referred to as the hospital “doom loop”.

---

## Dashboard Features

The dashboard includes interactive visualisations for major model variables, including:
- Elective Waiting List
- Acute Inpatients
- Discharge-Ready Patients
- Burnout
- Occupied Beds

Additional functionality includes:
- Run simulation controls
- Pause simulation controls
- Reset simulation controls
- Live simulation-day tracking
- Browser-based interactive graph rendering

The graphical presentation intentionally mirrors standard Vensim output conventions to preserve consistency with the original SD model implementation.

---

## Simulation Horizon

The dashboard reproduces the simulation horizon used in the underlying Vensim model:

- **Simulation Length:** 4,015 days
- **Time Unit:** Days
- **Simulation Method:** Discrete daily time-step integration

## Technical Requirements

The dashboard runs entirely client-side within a web browser.

### Recommended Browsers
- Google Chrome
- Microsoft Edge
- Mozilla Firefox
- Safari

### Dependency

The dashboard uses Plotly.js via CDN:

```html
<script src="https://cdn.plot.ly/plotly-latest.min.js"></script>
```

No server-side configuration is required.

## Usage

1. Download or clone the repository
2. Open the HTML dashboard file in a modern web browser
3. Press **Run** to begin the simulation
4. Use **Pause** or **Reset** as required

The dashboard may also be hosted using:
- GitHub Pages
- Institutional repositories
- Standard web hosting services

## Disclaimer

This dashboard is intended for research, educational, and illustrative purposes only and should not be interpreted as an operational forecasting, healthcare-planning, or decision-support tool. The outputs presented are derived from a simplified System Dynamics representation designed for academic exploration and communication of feedback-driven hospital congestion mechanisms. The model does not represent any specific hospital, health service, or national healthcare system and should not be used for clinical, operational, financial, or policy decision-making without further validation and contextual analysis.
